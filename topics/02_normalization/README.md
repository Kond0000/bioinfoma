# 02 Normalization

## このトピックでやること

サンプルごとの総カウント数の差を補正し、比較しやすい値に変換します。

## 目的

- なぜ正規化が必要か理解する
- CPMやlog変換の意味をつかむ
- PCAやヒートマップの前処理を準備する

## 具体的に役立つ場面

炎症刺激群の総read数が対照群より多いときに、「見かけ上高いだけなのか、本当に発現が高いのか」を切り分ける場面で役立ちます。

## このトピックで解く課題

「IL6 が刺激群で高いのは、単にたくさん読まれたからなのか、それとも正規化しても高いのか」を確かめるために、CPM と log 変換を使います。

## 教材の内容

- ライブラリサイズの違いが比較をゆがめる理由
- CPMでスケールをそろえる考え方
- `log2(x + 1)` で分布を扱いやすくする意味

## ゴール

生の count をそのまま比較してはいけない理由を、自分の言葉で説明できることです。
