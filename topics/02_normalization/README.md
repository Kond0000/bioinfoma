# 02 Normalization

## この章の問い

`IL6` が刺激群で高く見えるとき、それは本当に発現が高いのでしょうか。  
それとも、刺激群のサンプルでたまたま多くのreadが読まれただけなのでしょうか。

この章では、RNA-seq解析で生の count をそのまま比較してはいけない理由を学びます。

## なぜ正規化が必要か

RNA-seqでは、サンプルごとに読まれるRNA断片の総数が同じとは限りません。あるサンプルでは全体で1,000万read、別のサンプルでは2,000万readが得られることがあります。

このとき、同じ遺伝子でも、たくさん読まれたサンプルでは count が大きく見えやすくなります。これはカメラで写真を撮るときに、明るい場所では全体が明るく写り、暗い場所では全体が暗く写るのに似ています。まず全体の明るさをそろえないと、対象そのものが明るいのか、撮影条件が違うだけなのかを判断できません。

RNA-seqの正規化は、この「サンプル全体の読まれ方の違い」を補正する処理です。

## この章で扱う考え方

- `library size`
  1つのサンプルで得られたread数の合計です。
- `CPM`
  counts per million の略で、100万readあたりの count にそろえる方法です。
- `log2(x + 1)`
  大きすぎる値の影響を弱め、PCAやヒートマップで扱いやすくする変換です。

## 具体的に役立つ場面

炎症刺激群の総read数が対照群より多いときに、「見かけ上高いだけなのか、本当に発現が高いのか」を切り分ける場面で役立ちます。

## この章で解き明かすこと

`IL6` が刺激群で高いのは、単にたくさん読まれたからなのか、それとも正規化しても高いのかを確かめます。

## 読み取れるようになること

この章を終えると、生の count、CPM、log-CPM の違いを説明できるようになります。また、RNA-seq解析で「正規化してから比較する」という手順がなぜ必要なのかを、データサイエンスの観点から説明できるようになります。
