# 06 Portfolio

## この章の問い

RNA-seq解析で得た結果を、他人が読んで理解できる形にするにはどうすればよいでしょうか。  
コードと図があっても、目的、方法、結果、解釈がつながっていなければ、ポートフォリオとしては伝わりません。

この章では、解析結果を GitHub 上で見せるための構成を学びます。

## なぜポートフォリオ化が必要か

研究や実務では、解析そのものと同じくらい「何をしたのかを説明する力」が重要です。第三者は、すべてのコードを最初から読んでくれるわけではありません。短時間で、解析の目的、データ、方法、結果、解釈が分かる構成にする必要があります。

良いポートフォリオは、学習ログではありません。読み手が「この人はデータをどう理解し、どの判断をして、どの結論に至ったのか」を追える資料です。

## この章で扱う考え方

- `解析目的`
  何を明らかにしたい解析なのかを一文で示します。
- `解析フロー`
  QC、正規化、探索的解析、差次的発現解析、機能解釈の順番を示します。
- `主要な図`
  PCA、ヒートマップ、volcano plot など、結論を支える図を選びます。
- `結果の要約`
  図や表から読み取ったことを、短い文章で説明します。

## 具体的に役立つ場面

自分の解析を指導教員、共同研究者、採用担当者に見せるときに、何を理解していて、どこまで自力で進められるかを短時間で伝える場面で役立ちます。

## この章で解き明かすこと

このRNA-seqケーススタディで何が分かり、なぜその結論に至ったのかを、README と図だけで他人に伝わる形にまとめます。

## 読み取れるようになること

単なるノートブックの集まりではなく、研究課題、解析手順、結果、解釈が一続きになったポートフォリオを作れるようになります。
